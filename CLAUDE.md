# mipurin-adventure プロジェクトルール

## コンテキスト上限の管理

**セッションのコンテキスト使用量が +430 以上に近づいたら、以下を実行すること：**

1. 作業中の場合は、現在のタスクをいったん区切りの良いところで止める
2. 未コミットの変更があれば `git add` + `git commit` する
3. `git push -u origin <現在のブランチ名>` でプッシュする
4. ユーザーに以下のメッセージを伝える：

   > **⚠ コンテキストが +430 を超えました。新しいセッションを開いてください。**
   > プッシュ済み：`<ブランチ名>`
   > 新セッションで「続きから」と伝えると作業を引き継げます。

5. それ以上の作業は行わず、新セッションを待つ

## Git ブランチ運用

- 開発ブランチ：`claude/push-pending-changes-Q8DkQ`
- プッシュ先：`origin <ブランチ名>` を常に `-u` オプション付きで実行
