## 2. 実装済み機能 (v6.6.2)

### コアゲームプレイ
- WASD移動, Z攻撃, Xダッシュ（無敵フレーム付き）
- 武器6種（蜂の針/蜜砲/花粉盾/蔦鞭/羽根手裏剣/女王の杖）各固有エフェクト（360/AOE/double/pierce）
- 武器2本持ち（おきにいり/もうひとつ）, Qキー切替
- 消耗品3種（はちみつドロップ/ピリカラ花粉/ロイヤルゼリー）, 1/2/3キー使用
- ドロップ（花粉40%, 回復20%）, 磁力収集
- ヒットストップ（通常0.05s, ボス0.07s）, スクリーンシェイク, ノックバック
- パーティクルエンジン（命中/撃破/ダッシュ/祝福）

### 祝福システム
- 24祝福（6系統×4種: common/rare/rare/legend）
- 花の精霊6系統: ローザ(薔薇), リリア(百合), ソーレ(向日葵), フジカ(藤), ハスミ(蓮), キクネ(菊)
- デュオ祝福6種（2系統の組み合わせで自動発動）
- レアリティ加重選択（common:50, rare:30, legend:10）

### ダンジョン構造
- フロア間ノード選択画面（3択）: バトル/エリート/ショップ/休憩/イベント
- 5種部屋テンプレート（open/maze/circular/L_shape/cross）, テーマ別割り当て
- トゲ床（F4以降）, flood-fill経路保証
- 5テーマ（森/洞窟/花畑/深淵/遺跡）各固有BGM
- ウェーブ制（2〜5ウェーブ/フロア, 敵数フロアスケール）
- 3フロアごとにボス, 全15フロア

### 敵・ボス
- 敵12種（各: 日本語名, lore, 固有shape, AIパターン: wander/chase/charge/shoot/teleport）
- ボス4種（スズメバチの女王/キノコの王/クリスタルゴーレム/闇の蛾）各固有パターン, 2〜3フェーズ
- テーマ別敵プール, フロアスケーリング（HP/DMG/Score）
- ボス入場ダイアログ

### メタプログレッション
- ネクター（永続通貨: 敵撃破で蓄積, 死亡時保持）
- 花壇メニュー（HP +1 / ATK +1, 各5段階, コストエスカレーション）
- localStorage永続化

### ビジュアル
- mipurin.png 4方向スプライト + bob/squash/stretchアニメーション
- 敵12種 + ボス4種 WebPスプライト + 個別PNGスプライトエンジン
- 武器6 + 消耗品3 + ドロップ2 スプライト（全27枚）
- Canvas fallbackシェイプ（全12種固有描画）
- HiDPI対応（devicePixelRatio）, CSS responsiveキャンバス
- フェード演出, ダッシュトレイル, 攻撃スラッシュ弧

### オーディオ
- Web Audio合成 21種SE（attack, hit, hurt, kill, dash, blessing, clear, shop, buy, drop, game_over, boss_appear, item_get, level_up, door_open, menu_move, menu_select, dialog_open, dialog_close, player_hurt, enemy_die）
- BGM 5テーマ曲 + ボス曲 + タイトル曲（mp3, ループ再生）
- 状況連動（フロアテーマ切替, ボス戦切替, 死亡時停止）

### UI/UX
- 日本語UI全画面（タイトル/HUD/祝福/ショップ/花壇/インベントリ/図鑑/ダイアログ）
- 統一メッセージシステム（float型 + dialog型タイプライター）
- TABインベントリ（持ち物タブ + 図鑑タブ）
- 図鑑（敵撃破でlore解放）, 武器コレクション（localStorage）
- ショップ（2行×3列グリッド, 回復/消耗品/武器/最大HP）
- イベント2択システム（EVENT_POOL 5種）
- 操作ガイドバー, アイテムスロットUI, サブ武器インジケータ

### ストーリー
- プロローグ（画像10枚 + テキスト5行, Z送り/Xスキップ）
- ボス入場ダイアログ（4ボス各2行）
- フロア開始フロートメッセージ
- エンディング画面（F15ボス撃破後）

7. **ビジュアルポリッシュ**: 歩行アニメーション、環境エフェクト

## 7. 開発ルール（絶対遵守）

1. **忖度禁止**: 技術的に問題がある場合は理由を明示して率直に反対意見を述べる
2. **議論における対等性**: AIエージェントはプロジェクトの共同開発者として対等に議論する
3. **品質に対する妥協禁止**: 技術的負債を積む場合は内容・返済時期・リスクを必ず明記する
4. **根拠ベースの意思決定**: GDD v2.0のスコア表・コード品質指標など客観的根拠に基づく
5. **スコープ厳守**: Phase外の機能要望は明確に先送りする
