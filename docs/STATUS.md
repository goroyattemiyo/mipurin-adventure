## 2. 実装済み機能 (v6.11.3)

### コアゲームプレイ
- WASD移動, Z攻撃, Xダッシュ（無敵フレーム付き）
- 武器6種（蜂の針/蜜砲/花粉盾/蔦鞭/羽根手裏剣/女王の杖）各固有エフェクト（360/AOE/double/pierce）
- 武器2本持ち（おきにいり/もうひとつ）, Qキー切替
- 消耗品3種（はちみつドロップ/ピリカラ花粉/ロイヤルゼリー）, 1/2/3キー使用
- ドロップ（花粉40%, 回復20%）, 磁力収集
- ヒットストップ（通常0.05s, ボス0.07s）, スクリーンシェイク, ノックバック
- パーティクルエンジン（命中/撃破/ダッシュ/祝福）

### 祝福システム
- 78祝福（6系統×13種: common×8/rare×4/legend×1）
- 花の精霊6系統: ローザ(薔薇), リリア(百合), ソーレ(向日葵), フジカ(藤), ハスミ(蓮), キクネ(菊)
- デュオ祝福15種（2系統の組み合わせで自動発動）
- レアリティ加重選択（common:50, rare:30, legend:10）

### ダンジョン構造
- 2段ミニツリー型ノードマップ（分岐・収束）: バトル/エリート/ショップ/休憩/イベント/宝箱/ミステリー/ボーナス
- 5種部屋テンプレート（open/maze/circular/L_shape/cross）, テーマ別割り当て
- トゲ床（F4以降）, flood-fill経路保証
- 5テーマ（森/洞窟/花畑/深淵/遺跡）各固有BGM
- ウェーブ制（2〜5ウェーブ/フロア, 敵数フロアスケール）
- 3フロアごとにボス, 全15フロア

### 敵・ボス

- ボス4種（スズメバチの女王/キノコの王/クリスタルゴーレム/闇の蛾）各固有パターン, 2〜3フェーズ

- ボス入場ダイアログ

### メタプログレッション

- 花壇メニュー（HP/ATK/速度/ダッシュCD/磁力/ネクター倍率 6種, クリア数アンロック）

## 7. 開発ルール（絶対遵守）

1. **忖度禁止**: 技術的に問題がある場合は理由を明示して率直に反対意見を述べる
2. **議論における対等性**: AIエージェントはプロジェクトの共同開発者として対等に議論する
3. **品質に対する妥協禁止**: 技術的負債を積む場合は内容・返済時期・リスクを必ず明記する
4. **根拠ベースの意思決定**: GDD v2.0のスコア表・コード品質指標など客観的根拠に基づく
5. **スコープ厳守**: Phase外の機能要望は明確に先送りする

### ファイル管理ルール
6. **ファイルサイズ上限**: 各JSファイルは30KB以下を維持する
7. **分割トリガー**: 25KBを超えたら分割を検討、30KBを超えたら即分割
8. **結合禁止**: 分割済みファイルの再結合はCOUNCIL審議を経ること
9. **index.html読み込み順**: game.js → data.js → enemies.js → blessings.js → systems.js → nodemap.js → ui.js → ui_screens.js → update.js → render.js
10. **キャッシュバスター**: JS変更時は index.html の ?v= パラメータを必ず更新する

### ワークフロールール
11. **ファイル参照ルール**: push後はリポジトリ（GitHub raw/API）から直接ファイルを参照する。ユーザーにファイル検索（Select-String等）を依頼できるのは編集後push前の検証時のみ
12. **フェーズ遷移ルール**: 次のSprintに進む際は、リポジトリから最新コードを全取得し現状を把握した上で、最高有識者（該当分野の専門家ペルソナ）を召喚して設計議論を行ってから実装に入る
