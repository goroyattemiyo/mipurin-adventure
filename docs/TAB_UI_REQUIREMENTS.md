# TAB_UI_REQUIREMENTS.md — TABメニューUI改修 要件定義 v0.2（仮）

対象: ミプリンの冒険 / TABメニューUI改修

---

## 1. 背景

現行のTABメニューは「持ち物 / 図鑑 / 装備」の3タブ構成で、PC版ではキー操作、スマホ版ではタッチUIと仮想ボタンで操作が分かれている。
図鑑・装備・スマホ対応はすでに一定の実装資産があるため、本改修は全面刷新ではなく、既存資産を活かしながら、可読性・非重複・かわいさを高める改修とする。

## 2. 改修目的

本改修の目的は、TABメニューUIを「開いた瞬間に分かる」「重ならない」「ミプリンらしくかわいい」方向へ整理することにある。

特に重視する課題は以下の通り。

- スマホ版での文字の小ささ
- アイテム名や装備名の判読性
- UI要素同士の重なり
- PC/スマホで意味は共通、操作は別の整理
- 主人公ミプリンのかわいさを活かした装備UI

## 3. 前提条件

### 3-1. 操作前提
- PC版とスマホ版で操作方法は統一しない
- PC版はキーボード中心
- スマホ版はタップ中心
- ただし、各タブの役割と情報構造は共通とする

### 3-2. UI起動条件
- TABメニューは playing / waveWait 中のみ開けるUIとする
- 祝福、ショップ、イベント、ダイアログ等との同時表示は行わない

## 4. 最重要要件

### 4-1. 非重複要件
TABメニュー内では、文字、アイコン、ボタン、説明文、ヘルプ導線、キーヒントが相互に重なってはならない。
特にスマホ版では、情報量よりも非重複を優先する。

対象:
- タブ見出しとヘルプ / 閉じる導線
- ステータス文字と装備サマリー
- アイテム枠とラベル文字
- 祝福名と説明文
- 本文とスマホの画面ボタン
- 本文とPC用キーヒント

### 4-2. 可読性要件
表示される文字は、読めることを合格条件とする。
スマホ版では、PC版と同じ情報量を維持することより、判読性を優先する。

最小文字サイズの目安:
- 見出し: 20px以上
- ラベル: 16px以上
- 補助説明: 14px以上

これ未満になりそうな場合は、縮小で押し込まず、
再配置 → 改行 → 省略 → 詳細表示へ退避
の順で対応する。

### 4-3. 自動調整要件
UIは画面サイズと入力方式に応じて、各情報ブロックの配置・表示密度・表現形式を自動調整すること。
この調整はフォント縮小のみで対処してはならず、優先的にレイアウトの切り替えで対応する。

## 5. レイアウト方針

### 5-1. 安全領域ベース
TABメニューは、画面全体に直接固定座標で置くのではなく、次の領域に分けて構成する。

- ヘッダー領域
- 本文領域
- フッター領域
- スマホ版のタッチボタン除外領域

### 5-2. 情報ブロック方式
各タブの内容は、個別要素の直接描画ではなく、以下のような情報ブロックとして扱う。

- ステータスブロック
- 装備サマリーブロック
- アイテムブロック
- 祝福ブロック
- 詳細説明ブロック
- 所持一覧ブロック
- 比較 / 強化ブロック

各ブロックは、理想サイズ・最小サイズ・省略時表示・スマホ時の簡易表示を持つ前提で設計する。

## 6. 共通UI要件

### 6-1. ヘッダー
全タブに共通して、上部に以下を配置する。

- タブ名
- タブ切替導線
- ヘルプアイコン
- 閉じる導線

ヘルプは各タブごとに文言を切り替えるが、表示位置と開閉方法は統一する。

### 6-2. フッター
下部には操作ガイドを表示する。
ただし、本文領域を侵食しないことを前提とする。

- PC版: キー操作ヒント
- スマホ版: ボタン / タップヒント

## 7. 持ち物タブ要件

### 7-1. 役割
持ち物タブは、現在のプレイヤー状態をひと目で把握するダッシュボードとする。

### 7-2. 必須表示項目
- HP
- ATK
- 速度
- フロア
- スコア
- 花粉
- 現在武器
- サブ武器
- チャーム
- 消耗品3枠
- 祝福サマリー
- 選択中要素の詳細欄

### 7-3. 表示方針
PC版:
- 左: ステータスカード群
- 右: 装備サマリー
- 下: アイテムと祝福
- 最下部: 詳細欄

スマホ版:
- 上: ステータス
- 中: 装備サマリー
- 下: アイテム
- 下: 祝福要約
- 最下部: 詳細欄

### 7-4. 表現方針
- ステータスは縦列テキストではなく、小カード化する
- 長文祝福説明は常時表示せず、要約または詳細欄送り
- アイテム枠は文字よりアイコン優先
- 名前や説明が長い場合は、縮小よりも改行・省略で対処する

## 8. 図鑑タブ要件

### 8-1. 役割
図鑑タブは、収集状況の確認と閲覧を楽しむ画面とする。

### 8-2. 維持する構造
以下は既存資産として原則維持する。

- サブタブ
- フィルタ
- 進捗表示
- カルーセル
- 詳細ポップアップ

### 8-3. 改善対象
- サブタブの見やすさ
- フィルタの押しやすさ
- 現在位置の分かりやすさ
- スマホにおけるスワイプとタップの競合回避
- 未発見要素の魅力的な見せ方

## 9. 装備タブ要件

### 9-1. 役割
装備タブは、主人公に装備を身につけさせている感覚を見せる画面とする。
ただし、システム上の意味は既存の メイン / サブ / チャーム を維持する。

### 9-2. 基本構成
- 中央: 主人公ミプリン
- 周囲: メイン / サブ / チャームの装着スロット
- 右または下: 所持一覧
- 下部: 詳細・比較・強化 / 進化情報

### 9-3. デザイン方針
- 主人公が主役に見えること
- 装備中のものが視覚で分かること
- 一覧性と比較性を損なわないこと
- きせかえ感と判断しやすさを両立すること

## 10. 将来拡張要件

### 10-1. 武器バリアント
既存武器を基礎単位とし、その上に色違い・テーマ違い・見た目違いのバリアントを追加して、収集要素を拡張する。
主目的は装備更新効率ではなく、発見・収集・見た目の変化を楽しめるコレクション性の向上に置く。
なお、G1 / G2 / G3 のような別概念とは混同せず、本作では採用しない。

### 10-2. おしゃれ装着品
防具のような重い概念は採用せず、アクセサリ系・みつばち寄りのお洋服系を「おしゃれ装着品」として将来的に追加する方向を想定する。
これは戦闘装備とは分離し、主に見た目とコレクション性を高める目的で導入する。

想定カテゴリ:
- あたま
- からだ
- かざり

## 11. 非機能要件
- PC / スマホ両方で、主要文字が肉眼で判読可能であること
- スマホ横画面で、本文がタッチボタンと重ならないこと
- 長い名称や説明文を、極小フォントで無理に詰め込まないこと
- UIは常に1手で戻れること
- 今どこが選択中か が常に明確であること

## 12. 改修優先順位
1. 共通レイアウトルール整備
2. 持ち物タブ再設計
3. 装備タブの主人公中心化
4. 図鑑タブの見やすさ調整
5. 将来拡張として武器バリアント / おしゃれ装着品

## 13. 今回やらないこと
- PC / スマホの操作統一
- 図鑑構造の全面刷新
- 武器バリアントの本実装
- 防具システムの導入

## 14. 模擬有識者会議レビュー（2026-03-29）

TAB UI要件定義 v0.2 について、以下の想定ユーザー視点で模擬レビューを実施した。

* 一般主婦ユーザー
* 高校生ユーザー
* ローグライト好きユーザー
* スマホUI重視ユーザー
* 実装・設計担当目線

### 14-1. 主な確認論点

* タブの役割が初見で分かるか
* スマホで文字が読めるか、重ならないか
* 装備タブの主人公中心化が有効か
* ミプリンらしいかわいさが出ているか

### 14-2. 合意できた点

* タブ構成は現状の「持ち物 / 図鑑 / 装備」のままでよい
* ただし「持ち物」は、単なるバッグ画面ではなく「現在の状態を見るダッシュボード」として見せる必要がある
* スマホ版は情報量より可読性を優先し、本文は安全領域内で縦積み中心に構成する
* 装備タブの主人公中心化は有効であり、ミプリンに装備している感覚を見せる方向を採用する
* ただし装備タブは見た目重視に寄りすぎず、一覧性・比較性・強化可否の判断しやすさを維持する
* かわいさは背景装飾を増やすより、主人公・装備アイコン・やわらかい演出で表現する
* 羊皮紙背景は全面背景ではなく、パネル背景や一部フレーム用途で検討する

### 14-3. 持ち物タブに関する示唆

* 最上段にステータスを置き、「いまのようす」が一目で伝わる構成が望ましい
* 武器・チャーム・アイテム・祝福は、長文を並べずに要約表示を基本とする
* 長い名前や説明は、常時表示ではなく詳細欄へ退避する
* スマホでは「全部見せる」より「読めること」を優先する

### 14-4. 装備タブに関する示唆

* 中央にミプリンを立たせ、周囲にメイン / サブ / チャームの装着スロットを配置する案は好評
* 見た目上は「きせかえ感」を強めつつ、システム上の意味は現行のメイン / サブ / チャームを維持する
* 所持一覧・比較欄・強化情報は引き続き必要であり、省略しない
* 将来的な武器バリアントやおしゃれ装着品の表示とも相性が良い

### 14-5. 図鑑タブに関する示唆

* 図鑑は既存構造を大きく変えず、見やすさ・押しやすさ・未発見要素の魅力的な見せ方を改善する方針が妥当
* 未発見はシルエットや `???` 表示を使い、収集欲を高める方向が望ましい

### 14-6. レビュー後の反映方針

* 持ち物タブはダッシュボード化を優先する
* 装備タブは主人公中心レイアウト案を採用候補とする
* スマホUIでは、安全領域・非重複・最小文字サイズ保証を最優先とする
* 羊皮紙素材は、可読性を損なわない範囲でパネル背景用途として検討する

### 15-7. 羊皮紙素材の利用方針（仮）

* `paper.webp` は全面背景ではなく、パネル背景や一部フレーム用途で検討する
* 想定用途:

  * 詳細欄の背景
  * 図鑑カードの背景
  * 持ち物タブの詳細説明パネル
  * 装備タブの比較欄背景
* 可読性を損なう場合は不採用とする
